The core paradox: Adding more predictors to a regression model can improve out-of-sample performance—even when the additional predictors are noisy—provided that (1) ridge regularisation is applied and (2) the true signal is diffuse across many weak predictors. This overturns the classical advice to keep models parsimonious. The optimal complexity ratio P/T is often 2–5×, not less than 1.
Practitioner implication: Risk teams should not reflexively shrink their feature sets. The question is not “how few features can we use?” but “with what regularisation strength should we use all of them?” The Sharpe Ratio, not R², should be the primary evaluation metric.

Three regimes: When c < 1 (under-parameterised), OLS works but R² degrades as c approaches 1. At c = 1 (interpolation threshold), the Gram matrix X'X becomes singular and OLS explodes—R² collapses to negative infinity and coefficient norms diverge. When c > 1 (over-parameterised), OLS literally cannot be computed (more parameters than observations). This “c = 1 crisis” is not a curiosity—it is the regime many quantitative teams operate in.
Danger zone: OLS is fundamentally broken when P ≈ T. Any portfolio strategy relying on unregularised regression with a complexity ratio near 1 is fitting noise. The coefficient norm explosion means the strategy is taking massive, economically meaningless positions.

The key discovery: Ridge regression (adding λI to X'X) eliminates the c = 1 singularity. It works in all three regimes—including c > 1 where OLS is undefined. Moreover, with optimal penalty z*, the ridge estimator achieves higher Sharpe Ratio at c = 2 than OLS achieves at c = 0.5. More parameters + regularisation > fewer parameters + no regularisation.
Critical insight: Negative R² is compatible with positive Sharpe Ratio. A model can have poor mean-squared-error performance (bad R²) yet produce profitable portfolios (good SR) because the Sharpe Ratio depends on the sign of the prediction, not its magnitude. This is why R² alone is a misleading metric for evaluating return-prediction models.

Module 3: The Marchenko-Pastur Distribution

Eigenvalue Distribution vs c

MP Support Bounds

Explore: Concentration Ratio

0.50
Why eigenvalues spread: Even when the true covariance is the identity matrix (Σ = I, all eigenvalues equal 1), the sample eigenvalues spread over the interval [(1 − √c)², (1 + √c)²]. This is pure estimation noise. As c → 1, the smallest eigenvalue approaches zero, making the sample covariance singular. The Marchenko-Pastur law quantifies exactly how much spreading to expect.
Practical consequence: The condition number of the sample covariance κ = λmax/λmin grows explosively as c → 1. At c = 0.9, κ ≈ 361. This means small estimation errors in expected returns are amplified 361× in the portfolio weights. This single fact explains most MVO failures in practice.

The mathematical engine: The Stieltjes transform m(−z; c) encodes the entire eigenvalue distribution of the sample covariance into a single analytic function. For Ψ = I, it has the closed form: m = [−(1−c+z) + √((1−c+z)² + 4cz)] / (2cz). The bridge function ξ(z; c) = (1 − z·m) / (c¹ − 1 + z·m) then maps directly to all portfolio performance metrics: expected return, leverage, R², and Sharpe Ratio.
Why this matters: Without the Stieltjes transform, the performance of a ridge regression portfolio with P predictors and T observations would require Monte Carlo simulation. With it, we get exact analytical formulas. This transforms the problem from “guess and check” to “compute the optimal penalty in closed form.”

Propositions 3–4: All portfolio metrics are determined by the single function ξ(z;c). Expected return E = b*·ξ, Leverage L = ξ/c, R² = b*·ξ²/c, Sharpe = b*·ξ/√(c·L). The optimal ridge penalty is z* = c/b*, which means stronger signal (higher b*) requires less shrinkage, and more complexity (higher c) requires more shrinkage. This is the quantitative prescription for how to set regularisation.

Bias-variance mechanics: Increasing the ridge penalty z reduces variance (more stable estimates) but increases bias (estimates shrunk toward zero). The MSE-optimal penalty balances these two forces. However, the Sharpe-optimal penalty differs from the MSE-optimal one because leverage falls faster than expected return with increasing z. A model can have high MSE but excellent risk-adjusted performance if the shrinkage reduces portfolio leverage disproportionately.
The c = 1 crisis in detail: At the interpolation threshold, variance dominates: the model fits noise perfectly in-sample but produces nonsensical out-of-sample predictions. Ridge regression rescues this by adding a diagonal “cushion” that prevents inversion of near-zero eigenvalues. The required penalty strength z* increases with c, which is why the optimal ridge penalty is z* = c/b*.

Module 7: Model Misspecification & Theorem 1

SR vs Complexity: Ridgeless vs Optimal Ridge

R² vs Complexity: Ridgeless vs Optimal Ridge

Theorem 1 (The Virtue of Complexity): Under optimal ridge regularisation, both the Sharpe Ratio and R² are strictly increasing and concave in model complexity q = P/Ptrue. Adding more predictors always helps, provided that (a) the new predictors carry even infinitesimal signal and (b) the ridge penalty is adjusted optimally. This is the paper's central result.
Without regularisation, the story reverses: The ridgeless (OLS) estimator shows a “double descent” pattern—performance degrades near c = 1, partially recovers beyond it, but never reaches the level achievable with optimal ridge. The gap between the two curves is the value of regularisation. Misspecification (using the wrong model) reduces the slope of the ascent but does not reverse it: complexity still helps.

Module 8: Random Fourier Features

Approximation Quality vs Feature Count

SR vs Feature Count (Multiple Bandwidths)

From linear to non-linear: Random Fourier Features transform K raw predictors into P non-linear features via Si,t = [sin(γω'G), cos(γω'G)], where ω is drawn randomly and γ controls the bandwidth. This approximates a Gaussian kernel and allows ridge regression to capture non-linear relationships. The “virtue of complexity” extends: more RFF features (higher P) improve out-of-sample performance with optimal ridge, achieving SR ≈ 0.47 on realistic synthetic data.
Bandwidth γ matters: Too small a γ produces near-linear features (no benefit over raw predictors). Too large a γ creates features that vary too rapidly and fit noise. The optimal γ depends on the scale of the underlying signal. In practice, cross-validation over γ jointly with the ridge penalty z gives the best results.
